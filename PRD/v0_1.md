# FlowDay — Product Requirements Document

**A visual daily task flow planner with Todoist integration**
*Spiritual successor to HourStack, rebuilt for the modern solo productivity stack*

---

## 1. Problem Statement

HourStack was acquired by ClickUp and discontinued as a standalone product. It left behind a gap for users who loved its core workflow: **pull tasks from Todoist → visually arrange them into a day plan → execute sequentially → track what actually happened**. No current tool replicates this well. Sunsama is close but costs $16/mo and overcomplicates things. What's needed is a lean, beautiful, open-source alternative you fully own.

### The Core Insight (What Made HourStack Special)

Most time-blocking apps force you to assign exact clock times to tasks. But knowledge work doesn't work that way — you don't know if the 2pm meeting will run long, or if you'll get into flow state on the morning coding task. **FlowDay embraces sequential ordering over clock-time scheduling.** You decide *what* to do today and *in what order*, then simply work through the list. The app tracks how long each task actually takes, building data over time.

---

## 2. Target User

Solo knowledge workers (developers, designers, writers, consultants) who already use Todoist as their task inbox and want a daily execution layer on top. Not teams — keep it personal and fast.

---

## 3. Core Concepts

| Concept | Description |
|---------|-------------|
| **Task Pool** | Sidebar showing Todoist tasks (today's tasks, a project, a filter). This is your "available work" drawer. |
| **Day Flow** | The main canvas — an ordered list of tasks you've committed to today. Not time-slotted, just sequenced. |
| **Active Task** | The task you're currently working on. One at a time. Timer runs automatically. |
| **Time Entry** | A recorded span: which task, start time, end time, duration. Builds your history. |
| **Day View / Multi-Day View** | Toggle between focusing on today vs. seeing a 3-day or week-long horizon. |

## 4. Feature Breakdown

### 4.1 — Todoist Sidebar (Task Pool)

**What it does:** Syncs with Todoist and shows tasks in a collapsible sidebar panel.

- Authenticate via Todoist OAuth (or personal API token for MVP)
- Display tasks grouped by: Today, Overdue, a specific Project, or a custom Filter
- Each task card shows: title, priority color, project name, labels, due date
- Drag a task from the sidebar → drop into the Day Flow
- When dragged to Day Flow, the task is *linked* (not duplicated) — completing it in FlowDay marks it done in Todoist
- "Refresh" button + auto-sync every 60 seconds
- Search/filter within the sidebar

**API notes:** Use the new Todoist Unified API v1 (`/api/v1/tasks`). The official TypeScript SDK (`@doist/todoist-sdk-typescript`) is maintained by Doist and supports the v1 API. The old REST v2 and Sync v9 are deprecated as of early 2026.

### 4.2 — Day Flow (Main Canvas)

**What it does:** An ordered, drag-reorderable list of tasks for the day.

- Drag to reorder tasks within the flow
- Each task card shows: title, estimated duration (user-editable), priority indicator, actual time (once started/completed)
- Click a task → it becomes the **Active Task**, timer starts
- "Complete" button → stops timer, logs the time entry, marks task done (syncs to Todoist)
- "Skip" → moves task to bottom or to tomorrow
- "Pause" → pauses timer, no task is active
- Visual progress bar: how many tasks done vs. total, total time planned vs. elapsed
- End-of-day: any unfinished tasks can be bulk-moved to tomorrow

**Key UX detail:** The flow should feel like a stack. The top task is "next up." When you finish one, the next auto-highlights. This removes the friction of deciding what to do next.

### 4.3 — Time Tracking (Passive & Automatic)

- Timer starts when you activate a task, stops when you complete/pause/switch
- Manual time entry editing (in case you forgot to start/stop)
- Daily total tracked time shown prominently
- No Pomodoro enforcement — just a running clock. Optional break reminders.

### 4.4 — Day View / Multi-Day View

**Single Day View:**
- Full focus on today's flow. Large task cards, prominent timer, clear "what's next" indicator.

**Multi-Day View (3-day or 5-day):**
- Horizontal layout: columns for each day (like a Kanban-ish calendar)
- Drag tasks between days for quick rescheduling
- Each day column shows: task list, total estimated time, total actual time
- Past days are slightly dimmed, show completed status
- Useful for weekly planning on Sunday/Monday

**Toggle:** Prominent switch in the top nav. Keyboard shortcut: `1` for single, `3` for 3-day, `5` for week.

### 4.5 — Analytics Dashboard

**Daily Review Panel (end of day):**
- Planned vs. actual time per task (bar chart comparison)
- Tasks completed vs. skipped vs. rolled over
- Total productive time
- "How was your day?" quick mood/energy rating (1-5 scale, optional)

**Weekly/Monthly Reports:**
- Time distribution by Todoist project (donut chart)
- Estimation accuracy trend: are you getting better at estimating? (line chart over weeks)
- Productivity heatmap: which days/hours you're most productive
- Streak tracker: consecutive days of completing your flow
- Average tasks completed per day

**Insights (stretch goal):**
- "You consistently underestimate design tasks by ~40%"
- "Your most productive day is Tuesday"
- "You complete 73% of tasks you plan"

### 4.6 — Additional Features

- **Quick Add:** `Cmd+N` to add a local task (doesn't need to come from Todoist)
- **Color Coding:** Tasks inherit Todoist project colors. Custom color overrides available.
- **Keyboard Navigation:** `Space` to start/stop timer, `Enter` to complete, `↑/↓` to navigate, `D` to toggle done
- **Dark Mode / Light Mode:** System-preference aware, manual toggle
- **Local-first data:** Time entries and flow history stored locally. Todoist is source of truth for tasks, FlowDay is source of truth for time data.
- **Export:** CSV/JSON export of time entries for invoicing or spreadsheet analysis

---

## 5. What FlowDay is NOT

- Not a full project management tool (Todoist handles that)
- Not a team collaboration tool (no shared views, no resource allocation)
- Not a calendar app (no clock-time slots, no meeting integration in v1)
- Not a Pomodoro app (continuous tracking, not intervals)

## 6. Tech Stack Recommendation

### Why Next.js + Local-First

For vibe coding, you want fast iteration, good DX, and a polished result with minimal infrastructure headaches. Here's the recommended stack:

### Frontend

| Layer | Choice | Why |
|-------|--------|-----|
| **Framework** | **Next.js 15 (App Router)** | File-based routing, RSC for fast loads, great ecosystem. Vercel deploys in seconds. |
| **UI Library** | **shadcn/ui + Tailwind CSS** | Copy-paste components, fully customizable, looks professional out of the box. No vendor lock-in. |
| **Drag & Drop** | **@dnd-kit** | Modern, accessible, performant DnD library. Works great for both sidebar→flow and reordering. |
| **State Management** | **Zustand** | Lightweight, no boilerplate. Perfect for managing flow state, timer, sync status. |
| **Charts** | **Recharts** | React-native charting. Good enough for bar charts, line charts, donuts. |
| **Icons** | **Lucide React** | Clean, consistent icon set. Already pairs with shadcn/ui. |
| **Date/Time** | **date-fns** | Tree-shakeable, functional API. Better than moment/dayjs for this use case. |

### Backend & Data

| Layer | Choice | Why |
|-------|--------|-----|
| **Database** | **SQLite via better-sqlite3** (or **Turso** for cloud) | Local-first, zero config. Time entries don't need a cloud DB. Turso if you want cross-device sync later. |
| **ORM** | **Drizzle ORM** | Type-safe, lightweight, works great with SQLite. Better DX than Prisma for small projects. |
| **API Layer** | **Next.js Route Handlers** | `/app/api/*` routes for Todoist proxy and local data CRUD. No separate backend needed. |
| **Auth (Todoist)** | **OAuth 2.0 flow** (or API token input for MVP) | Full OAuth for production, but start with a simple token input to move fast. |
| **Hosting** | **Vercel** (or self-host with Docker) | Free tier is generous. Or run locally with `npm run dev` forever. |

### Alternative: Electron Desktop App

If you prefer a native desktop experience (like HourStack was web-only but felt native):
- Use **Tauri 2.0** instead of Electron (Rust-based, much smaller binary)
- Same React/Next.js frontend, but bundled as a desktop app
- SQLite is even more natural in a desktop context
- Tray icon with timer display

## 8. Database Schema

```sql
-- Daily flow: which tasks are planned for which day, in what order
CREATE TABLE flows (
  id            TEXT PRIMARY KEY,
  date          TEXT NOT NULL,          -- '2026-04-12'
  task_order    TEXT NOT NULL,          -- JSON array of task IDs (ordered)
  created_at    TEXT DEFAULT (datetime('now')),
  updated_at    TEXT DEFAULT (datetime('now'))
);

-- Individual tasks (local cache + local-only tasks)
CREATE TABLE tasks (
  id              TEXT PRIMARY KEY,
  todoist_id      TEXT,                 -- NULL for local-only tasks
  title           TEXT NOT NULL,
  project_name    TEXT,
  project_color   TEXT,
  priority        INTEGER DEFAULT 1,
  labels          TEXT,                 -- JSON array
  estimated_mins  INTEGER,             -- User's time estimate
  is_completed    INTEGER DEFAULT 0,
  completed_at    TEXT,
  created_at      TEXT DEFAULT (datetime('now'))
);

-- Time entries: the core data FlowDay owns
CREATE TABLE time_entries (
  id          TEXT PRIMARY KEY,
  task_id     TEXT NOT NULL REFERENCES tasks(id),
  flow_date   TEXT NOT NULL,           -- Which day this entry belongs to
  start_time  TEXT NOT NULL,           -- ISO timestamp
  end_time    TEXT,                    -- NULL if still running
  duration_s  INTEGER,                -- Computed on stop, in seconds
  created_at  TEXT DEFAULT (datetime('now'))
);

-- Daily reflections (optional analytics)
CREATE TABLE daily_reviews (
  id          TEXT PRIMARY KEY,
  date        TEXT NOT NULL UNIQUE,
  mood_rating INTEGER,                -- 1-5
  energy_rating INTEGER,              -- 1-5
  notes       TEXT,
  created_at  TEXT DEFAULT (datetime('now'))
);
```

## 10. Vibe Coding Plan — Build Order

The key to vibe coding this successfully is building in **vertical slices** — each milestone is a usable, demoable app. Don't build all components first then wire them up.

### Phase 1: The Core Loop (Days 1-3)
> Goal: You can drag tasks into a day and work through them with a timer.

1. Scaffold Next.js + Tailwind + shadcn/ui
2. Build the **App Shell** (sidebar + main canvas layout)
3. Create **mock task data** (hardcoded array, no Todoist yet)
4. Build **Task Pool sidebar** with mock tasks
5. Build **Day Flow** with drag-and-drop (dnd-kit)
6. Implement **Timer** (start/stop/complete on a task)
7. Wire up **SQLite** + Drizzle for time entries
8. Make it look good — this is where you spend extra time on polish

### Phase 2: Todoist Integration (Days 4-5)
> Goal: Real tasks from your actual Todoist account appear and sync.

1. Set up Todoist API token input (settings page)
2. Build API route to fetch today's tasks from Todoist
3. Replace mock data with real Todoist tasks in the sidebar
4. Implement "complete" sync — completing in FlowDay → completes in Todoist
5. Add sync indicator and refresh button
6. Handle edge cases: task edited in Todoist, task deleted, etc.

### Phase 3: Multi-Day & Polish (Days 6-7)
> Goal: Plan your week, not just today.

1. Build **Multi-Day View** (3 columns, drag between days)
2. Date navigation (prev/next day, date picker)
3. "Roll over" unfinished tasks to tomorrow
4. Keyboard shortcuts
5. Dark mode
6. Persistence: remember your flow when you close and reopen

### Phase 4: Analytics (Days 8-10)
> Goal: See where your time actually goes.

1. Build **Daily Review** panel (planned vs actual bar chart)
2. **Project breakdown** donut chart
3. **Estimation accuracy** trend line
4. **Productivity heatmap** (which hours you work most)
5. **Streak tracker**
6. Optional mood/energy daily rating

### Phase 5: Stretch Goals
- Google Calendar integration (show meetings as fixed blocks in day flow)
- Recurring flow templates ("my ideal Monday")
- AI-powered estimation suggestions based on history
- Mobile PWA support
- Webhook-based real-time Todoist sync
- Browser extension to quick-add tasks

## 11. Key Design Principles

1. **Sequential, not scheduled.** The day flow is an ordered list, not a calendar grid. This is the core differentiator.

2. **Todoist is the inbox, FlowDay is the workbench.** Don't rebuild task management. Let Todoist handle projects, priorities, recurring tasks. FlowDay just answers: "What am I doing today, and in what order?"

3. **Passive tracking.** The timer should be so frictionless that you forget it's running. Start on activate, stop on complete. That's it.

4. **Data compounds.** Every day you use FlowDay, your analytics get smarter. After a month, you'll know exactly how long "a code review" takes you.

5. **Beauty matters.** If the app doesn't look good, you won't open it. Invest in animations (Framer Motion), typography, spacing, and color. Study Linear, Raycast, and Arc for inspiration.

**Session 5 — Todoist Integration:**
> "Create a settings page with a text input for Todoist API token.
